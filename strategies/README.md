# 1.  Обзор системы

APEX V3.9.5 Strategy Tester — многоуровневая торговая система на Pine Script v6 для TradingView. Предназначена для торговли криптовалютными активами (прежде всего ETHUSDT) с использованием комплексного макроэкономического фильтра, технических индикаторов и автоматического протоколирования сделок.

Стратегия НЕ является сигнальным ботом и НЕ предназначена для полностью автоматической торговли. Главная цель — дать трейдеру объективную картину рыночных условий, отфильтровать опасные входы и зафиксировать все события в структурированном логе для последующего исследования через NotebookLM.

# 2.  Архитектура стратегии

Система построена по принципу последовательной фильтрации. Вход в позицию происходит только при одновременном выполнении всех активных фильтров. Каждый уровень независим и может быть включён или выключён отдельно.

## 2.1  Цепочка принятия решения

```
MACRO E1  →  IPDA ZONE  →  WEEKLY SWEEP  →  MACRO E2  →  ADX E3  →  RVOL E4  →  BTC E5
DXY/SPX      Discount/      Ликвидация      VIX/MOVE/    SLEEP/       Объём       BTC тренд
направление  Premium        недельного      JPY/EMA      TREND/       выше        и ETH/BTC
             зона           уровня          фильтр       OVER         нормы       ratio
```

Если хотя бы один активный фильтр не выполнен — сигнал не генерируется, событие записывается в диагностический лог как BLOCKED с указанием причины.

## 2.2  Описание блоков

### Блок E1 — Macro Triad (DXY, SPX, EMA Confluence)
Базовый макроэкономический фильтр. Определяет общее направление рынка.

- DXY (Индекс доллара США): если доллар слабее своей скользящей средней — условие для LONG выполнено
- SPX (S&P 500): если акции выше своей средней — рисковый аппетит есть
- EMA Confluence (опционально): требует выравнивания EMA20 < EMA50 для DXY и EMA20 > EMA50 для SPX
- macro_ok = dxy_bear AND spx_bull — главное булево условие системы

### Блок — IPDA Zones (Discount / Premium)
Определяет позицию цены относительно диапазона последних N баров.

- Lookback 20/40/60/80/100/200 баров — глубина исследуемого диапазона
- EQ (Equilibrium) = (High + Low) / 2 — середина диапазона
- **Режим REVERSAL:** Long в Discount, Short в Premium (SMC классика)
- **Режим TREND:** Long в Premium, Short в Discount (momentum вход)
- Позиция pos_pct (0–100%) записывается в каждую строку лога

### Блок — Weekly Sweep Detection
Детектор ликвидации недельных уровней.

- Берёт High и Low предыдущей полной торговой недели
- Если цена коснулась любого из уровней — weekly_sweep_hit = true
- Сбрасывается в false каждую новую неделю автоматически
- Обязательное условие: без sweep входа нет ни для LONG ни для SHORT

### Блок E2 — Macro Extended (VIX, MOVE, JPY)
Расширенный макрофильтр. Блокирует входы в периоды паники.

- VIX: при VIX > 20 статус WARN, при VIX > 35 статус PANIC (жёсткая блокировка)
- MOVE (ICE Bond Volatility): при MOVE > 140 рынок облигаций нестабилен
- JPY: резкое укрепление иены за 5 дней — бегство в защитные активы
- Статистически подтверждён: при VIX WARN win rate = 38% (13L/8W за 2023–2026)

### Блок E3 — ADX Engine (Sleep / Trend / Overheat)
Фазовый анализ тренда на основе реального DMI (ta.dmi в Pine v6).

- SLEEP (ADX < 20): консолидация — опциональная блокировка входов
- TREND (ADX 20–40): активный тренд — оптимальная фаза для входа
- OVER (ADX > 40): перегрев — опциональная блокировка
- ADX фаза влияет на ширину Stop Loss в режиме SL/TP = ADX

### Блок E4 — RVOL (Relative Volume)
- RVOL = текущий объём / средний объём за N баров
- На 4H неэффективен — использовать MOVE вместо
- На 1D рекомендованный порог: 1.2–1.5

### Блок E5 — BTC Correlation
- BTC Trend: BTC price > BTC SMA(24) — бычий тренд BTC подтверждён
- ETH/BTC Ratio: отношение ETH к BTC выше своей средней — ETH сильнее BTC
- На стадии сбора статистики. Оба переключателя OFF.

# 3.  Система управления рисками (SL/TP Engine)

## 3.1  Режим ATR (основной, рекомендуемый)

| Параметр | Описание |
|---|---|
| ATR SL mult (1.75) | Stop Loss = цена ± ATR × 1.75 |
| ATR TP1 mult (1.5) | Take Profit 1 = цена ± ATR × 1.5 |
| ATR TP2 mult (3.0) | Take Profit 2 = цена ± ATR × 3.0 (при Use TP2 = ON) |
| Use TP2 | 50% позиции закрывается на TP1, 50% на TP2 |

## 3.2  Режим VWAP
- SL = VWAP ± StdDev × 0.618
- TP = VWAP ± StdDev × 1.0

## 3.3  Режим ADX (адаптивный)
- SLEEP: ATR × 0.75 — узкий стоп
- TREND: ATR × 1.5 — стандартный стоп
- OVER: ATR × 2.5 — широкий стоп

# 4.  Система протоколирования (Log Export)

## 4.1  Торговые записи — log.info

```
date | L/S | entry | exit | profit% | bars_held | score |
vix  | vix_state | move | jpy% | macro | e2 | ipda% |
disc/prem | sweep | adx_phase | rvol | btc_state | ipda_mode | WIN/LOSS
```

| Поле | Значение |
|---|---|
| date | Дата закрытия (UTC, YYYY-MM-DD) |
| L/S | Long / Short |
| entry / exit | Цена входа и выхода |
| profit% | Прибыль/убыток в % |
| bars_held | Баров в позиции |
| score | Confluence Score 0–6 |
| vix / vix_state | Значение VIX и статус: OK / WARN / PANIC |
| move | Индекс MOVE |
| jpy% | 5-дневное изменение USD/JPY |
| macro / e2 | 1 = выполнен, 0 = нет |
| ipda% | Позиция цены: 0% = Low, 100% = High |
| disc/prem | DISC = Discount, PREM = Premium |
| sweep | 1 = sweep сработал |
| adx_phase | SLEEP / TREND / OVER |
| rvol | Относительный объём |
| btc_state | BTC_BULL / BTC_BEAR |
| ipda_mode | REVERSAL / TREND — режим при входе |
| WIN / LOSS | Итог сделки |

## 4.2  Диагностический аудит — log.warning

| Тип | Условие | Значение |
|---|---|---|
| BLOCKED_SWEEP | Macro OK, sweep = 0 | Недельный уровень не пробит |
| BLOCKED_E2 | Sweep OK, VIX/MOVE/JPY сработал | Рынок в режиме страха |
| BLOCKED_ADX | E2 OK, фаза SLEEP или OVER | Нет тренда или перегрев |
| BLOCKED_BTC | ADX OK, BTC медвежий | Крипторынок неблагоприятен |

```
1W (недельный):
  Свеча = 5 торговых дней
  Discount = многонедельный откат = реальная поддержка
  REVERSAL = классическая SMC логика ✅

1D (дневной):
  Свеча = 1 торговый день
  "Discount" на LB 40 = цена падала 2+ месяца
  В растущем рынке продолжает падать
  Premium = подтверждение направления = momentum ✅
```

# 6.  Возможности для исследования и отладки

## 6.1  Принцип A/B тестирования

Изменяется ровно один параметр, все остальные фиксируются.

**Пример: VIX фильтр (1D · REVERSAL · LB 40 · Jan 2023 – Mar 2026)**

| Конфигурация | Сделок | WR | PF | DD |
|---|---|---|---|---|
| VIX ON | 188 | 54.26% | 1.53 | 3.31% |
| VIX OFF | 215 | 51.63% | 1.229 | 3.59% |

Вывод: VIX заблокировал 27 сделок → +51,707 USDT. Среднее сохранение на блокировку: -1,915 USDT.

## 6.2  Workflow исследования через NotebookLM

- Шаг 1: Settings → Log Export → Enable Trade Log = ON
- Шаг 2: Дождаться пересчёта истории
- Шаг 3: Pine Editor → Logs → выделить всё → скопировать
- Шаг 4: Сохранить как .csv
- Шаг 5: Загрузить в NotebookLM
- Шаг 6: Задавать вопросы: «WR при VIX WARN?», «ipda_mode vs результат?»
- Шаг 7: Выключить Enable Trade Log = OFF

> ВАЖНО: лог включать только для сбора данных.

## 6.4  Порядок активации фильтров

- **Приоритет 1** — VIX = ON: подтверждён статистически. Включить сразу.

- **Приоритет 3** — ADX: после анализа BLOCKED_ADX записей
